# Configuration

Dédalo v7 uses a unified `.env` file for all configuration. Legacy `config.php` and `config_db.php` files are no longer used — they are automatically migrated to `.env` on first boot.

## Architecture

| File | Role |
| --- | --- |
| `/private/.env` | **User-editable configuration** — all entity-specific values (DB credentials, entity, langs, etc.) |
| `/private/.env.{hostname}` | Host-specific overrides (loaded first, takes priority over `.env`) |
| `config/bootstrap.php` | **Scaffolding only** — defines constants with safe defaults via `env_loader::get()`. Do NOT edit this file to change values. |
| `config/config_areas.php` | Area access control (allow/deny ontology tipos) |
| `config/config_core.php` | Installation state (managed by Dédalo, do not edit manually) |

### How it works

1. `bootstrap.php` loads `env_loader` (self-contained, no Dédalo constants needed)
2. `env_loader` reads `.env.{hostname}` then `.env` from `/private/`
3. `bootstrap.php` defines all constants using `env_loader::get('KEY', default)` — values from `.env` override defaults
4. Structural/computed constants (paths, derived values) are calculated automatically and **cannot** be set in `.env`

### Auto-migration

If `config/config.php` exists but `/private/.env` does not, `bootstrap.php` automatically runs `config_migrator` to generate the `.env` file. After successful migration and DB stability verification, old config files are moved to `/private/config_backup/` with timestamps.

Manual migration is also available:

```bash
php core/base/migrate_config.php              # Normal migration
php core/base/migrate_config.php --dry-run    # Preview without writing
php core/base/migrate_config.php --force      # Overwrite existing .env
```

## Setting up the .env file

### New installation

1. Create the `/private/` directory (sibling of `master_dedalo/`)

    ```shell
    mkdir -p /path/to/private
    ```

2. Copy the sample `.env` file

    ```shell
    cp private/sample.env private/.env
    ```

3. Edit the `.env` file with your installation values

    ```shell
    nano private/.env
    ```

4. Set file permissions (the file contains secrets)

    ```shell
    chmod 600 private/.env
    ```

### Multi-host setup

For installations serving multiple hostnames from the same codebase, create hostname-specific files:

```shell
private/.env.localhost      # for development
private/.env.myentity.org  # for production
private/.env               # fallback / shared values
```

`env_loader` loads `.env.{hostname}` first (first-wins principle), then `.env` for any remaining keys.

### Upgrading from v6 / legacy config

If you have existing `config/config.php` and `config/config_db.php` files, Dédalo will auto-migrate them on first boot. See [auto-migration](#auto-migration) above.

## .env file format

```ini
# Comments start with #
# Uncomment or add keys to override bootstrap.php defaults

DEDALO_ENTITY=my_entity
DEDALO_SALT_STRING=My_secure_Salt_String!_2046
DEVELOPMENT_SERVER=false

# Boolean values: true/false, yes/no, 1/0, on/off
IS_PRODUCTION=true

# Integer values
DEDALO_MAX_ROWS_PER_PAGE=10

# Array values use JSON
DEDALO_APPLICATION_LANGS={"lg-eng":"English","lg-spa":"Castellano"}

# Leave empty for null
DEDALO_SOCKET_CONN=
```

## Changing configuration values

1. Edit `/private/.env`

    ```shell
    nano /private/.env
    ```

2. Locate and change the KEY with the proper value

3. Save the file — changes take effect on next request

> Constants not present in `.env` fall back to their defaults defined in `bootstrap.php`. You only need to set values that differ from the defaults.
